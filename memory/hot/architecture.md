# Architecture (hot memory)

System structure, ownership, and data flows. Updated when these change.

## Project

`market_helper` is a broker-agnostic, **read-only** market research and
portfolio-analytics platform centered on IBKR. Produces position reports,
performance analytics (TWR/MWR, USD/SGD), risk reports, regime detection, and
a live NiceGUI dashboard. V1 is strictly read-only — no order placement.

## Layer ownership (source under `market_helper/`)

Dependency direction:
`cli → workflows → application → domain → data_sources / reporting / presentation`

- `application/portfolio_monitor/` — dashboard-triggered orchestration,
  artifact resolution, progress events, action contracts. CLI / workflows /
  dashboard share **one** input-contract layer at `application/portfolio_monitor/contracts.py`
  (9 `*Inputs` dataclasses incl. `BenchmarkRefreshInputs`).
- `domain/portfolio_monitor/services/` — reusable analytics: NAV/cashflow
  history, performance windows, risk math, Yahoo return cache, SPY/BIL
  benchmark attachment, FI vol, proxy logic.
- `domain/regime_detection/` — coordinator, layer-aware engine
  (`macro_nowcast` + `market_implied` active; ML layers gated).
- `domain/integration/` — cross-domain (stress tests, combined reports).
- `data_sources/` — IBKR (Client Portal, TWS/IB Gateway via `ib_async`, Flex),
  Yahoo, FRED adapters.
- `presentation/dashboard/` — live NiceGUI UI. **Interactive entry**; HTML is
  the deliverable. The dashboard embeds the rendered HTML in an iframe — no
  separate snapshot/Playwright pipeline is planned. A thin `shell.py` (chrome +
  cross-surface nav + `/` landing) wraps **two parallel surfaces**:
  `portfolio_monitor` (`/portfolio`) and `trade_advisor` (`/advisor`). Fixed
  pair, not a registry (ADR 0008); research/backtest/screener are out of scope.
- `reporting/` — HTML renderers (combined, performance, risk, regime).
- `regimes/`, `suggest/` — factor scoring + hysteresis; regime-to-policy
  suggestions.
- `portfolio/`, `backtest/`, `common/`, `cli/` — supporting layers.
- Legacy compatibility wrappers: `ui/`, `workflows/`, `utils/`, `config/`,
  `safety/`.

## Main data flows

1. **Position → Report**: IBKR snapshot → normalized models → CSV/HTML.
2. **Performance**: Flex XML + daily NAV snapshots →
   `nav_cashflow_history.feather` → MTD/YTD TWR/MWR.
3. **Risk**: Position CSV + Yahoo returns + proxy vols → portfolio vol,
   correlation, sector/country/duration breakdowns.
4. **Regime**: VIX / MOVE / HY_OAS / UST factor inputs + FRED macro panel →
   growth + inflation axes + risk overlay → concept-aggregated regime label.
5. **Dashboard**: Live TWS query → position snapshots → NiceGUI refresh.

## Canonical artifacts + config paths

- `configs/security_universe.csv` — **manually maintained** semantic universe.
- `data/artifacts/portfolio_monitor/security_reference.csv` — generated
  lookup cache (regenerated by tooling).
- `data/artifacts/portfolio_monitor/flex/nav_cashflow_history.feather` —
  daily NAV/cashflow store (canonical).
- `data/artifacts/portfolio_monitor/flex/performance_report_*.csv` — dated
  performance report.
- `configs/portfolio_monitor/report_config.yaml` — tracked risk/report
  runtime config.
- `configs/regime_detection/{fred_series,market_regime,regime_engine}.yml` —
  YAML-driven regime config (adding/retuning a signal never needs code).
- `data/external/regime_detection/historical/market_panel_to_2024.feather` —
  pre-2025 market panel baseline (~1 MB).

## Data layout

`data/{raw,interim,processed,cache,artifacts,external,research_artifacts}/`.
Internal intermediates use **Feather**; debug CSVs emitted on demand. Yahoo
return cache at `data/artifacts/portfolio_monitor/yahoo_returns/`.

## Config layout

`configs/{app,portfolio_monitor,regime_detection,integration}/`.
`configs/portfolio_monitor/local.env` is gitignored, local-only secrets.
